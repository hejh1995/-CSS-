（1）white-space 属性设置如何处理元素内的空白。normal（默认，空白会被浏览器忽略）。
pre（空白会被浏览器保留，其行为方式类似 pre 标签）。
nowrap（文本不会换行，直到遇到 br 标签为止）。
pre-wrap （保留空白符序列，但是正常地进行换行）。
pre-line （合并空白符序列，但是保留换行符）。
（2）text-overflow 属性规定当文本溢出包含元素时发生的事情。
 clip（修剪文本）。
ellipsis （显示省略符号来代表被修剪的文本）。
string （使用给定的字符串来代表被修剪的文本）。
（3）border-collapse 属性设置表格的边框是否被合并为一个单一的边框,collapse(合并为一个)，separent（边框被分开，默认值）
（4）tableLayout 属性用来显示表格单元格、行、列的算法规则。
对table元素或其他display：table样式的元素添加这个属性，为了确保这个属性发挥作用，需要给这些表格元素指定一个宽度（哪怕是100%）
automatic（默认。列宽度由单元格内容设定。）
fixed （列宽由表格宽度和列宽度设定。
如果不设定，则各列的宽度将是平均分配的，overflow和text-overflow属性都可以正常生效的，如果overflow的值为visible，表单的内容可能会溢出）
inherit （规定应该从父元素继承 table-layout 属性的值。）

5.垂直居中

(1)使用决定定位和位移

position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%);

绝对定位会脱离文档流；如果需要剧中的元素的高度超过视口，顶部就会被视口裁切掉；有些浏览器会导致元素的显示有一些模糊，可以使用transform-style：preserve-3d修复，但未来也不一定就没有问题

(2)使用视口单位vh

margin: 50vh auto 0; transform: translateY(-50%);

视口的不好演示哦，只适合在视口中居中的场景。

(3)使用flex

父元素设置如下时，子元素垂直居中。display: flex; align-items: center; justify-content: center;

6.紧贴底部的页脚

(1)使用min-height 控制中间的最小高度

内容部分设置最小高度，所以当内容很少时，页脚可以固定在底部

min-height: calc(100vh - 5em - 7em);5em 是页头的高度，7em是页脚的高度

它要求我们计算页脚、页头的高度，一但他们中的内容变化，可能引起高度的变化，就需要重新改变min-height的值。
